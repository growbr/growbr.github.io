Crie o seu primeiro Laboratório com o DVWA

Hoje vamos conhecer um projeto chamado Damn Vulnerable Web App ou mais conhecido como DVWA. Quem deve conhecer o DVWA ?

  • Profissionais de segurança
  • Desenvolvedores web
  • Professores
  • Estudantes

Algumas das falhas que podemos encontrar no DVWA.

  • Brute Force
  • Command Execution
  • CSRF
  • File Inclusion
  • SQLInjection
  • SQLInjection (bind)
  • Upload
  • XSS Reflected
  • XSS Stored

DVWA é uma aplicação web desenvolvida em PHP e MySQL (mais tambem funciona com o PostgreSQL)repleta de vulnerabilidades ,os objetivos do projeto é ajudar profissionais de segurança para testar suas habilidades e ferramentas em um ambiente controlado , ajudar desenvolvedores web a compreender melhor os processos de segurança de uma aplicações web e ajudar professores e estudantes em um ambiente controlado para estudos em sala de aula.

O objetivo do DVWA é praticar algumas das vulnerabilidades web mais comuns, com vários níveis de dificuldade, com uma interface simples e direta.

O Projeto é da RandomStorm e todo o codigo está no repositório do github junto com a documentação DVWA (versão 1.3) , na documentação você conhece melhor o DVWA.

Recomendações

Por ter diversas falhas de segurança não é recomendado instalar o DVWA em alguma hospedagem publica , ou algum tipo de servidor web em funcionamento pois isso pode causar muitas dores de cabeça , acredite!

Eu GreenMind recomendo fortemente o uso de uma maquina virtual de preferencia algum sistema linux rodando , eu uso o Debian e não tenho problemas !

Antes de instalar

Como já deve saber o DVWA é uma aplicação em PHP e Mysql (tambem funciona no postgreSQL) e o que precisamos ? Simplesmente um servidor web como o Apache ou o Nginx e o banco de dados Mysql instalado.

No windowns e possivelmente no Mac usaremos o XAMPP
